## Context

Расширение уже умеет:
- Отображать текущее время и длительность с миллисекундами (обновление каждые 10 мс).
- Копировать текущую временную метку кнопкой `.ytp-copy-time-btn`, вставленной после `.ytp-time-display`.
- Форматировать время через `formatVideoTime(seconds, showMs)`.
- Пересоздавать UI-элементы при навигации между видео через `handleNavigation()`.

Весь UI-код находится в `js/content.js` (IIFE), стили — в `styles.css`.

## Goals / Non-Goals

**Goals:**
- Добавить кнопки A/B в control bar без изменения архитектуры расширения.
- Показывать бейдж с интервалом прямо над control bar, вне зависимости от видимости панели.
- Маркеры на прогресс-баре с подсветкой сегмента.
- Клавиши `[` / `]` как альтернатива кнопкам.
- Полный сброс состояния при навигации.

**Non-Goals:**
- Сохранение точек A/B между сессиями (localStorage / storage).
- Несколько одновременных интервалов.
- Loop-режим воспроизведения сегмента.
- Поддержка Shorts (graceful skip как у кнопки копирования).

## Decisions

### 1. Размещение бейджа — `position: absolute` в контейнере видео

Бейдж вставляется в `.html5-video-container` или ближайший родитель `<video>` с `position: relative`. Альтернатива — вставить в `#movie_player` — менее стабильна при ресайзах.

**Почему:** Контейнер видео не скрывается вместе с control bar, поэтому бейдж всегда виден пока активен.

### 2. Состояние хранится в переменных модуля (IIFE-scope)

```js
let intervalStartTime = null;
let intervalEndTime   = null;
```

Альтернатива — объект состояния. Отклонено: избыточно для двух переменных; консистентно с `showMilliseconds`, `isVideoPlaying` и другими переменными расширения.

### 3. Маркеры — абсолютно позиционированные div внутри `.ytp-progress-bar`

Позиция: `left: (time / duration) * 100%`. Пересчитываются в `updateIntervalUI()`. Альтернатива — `<canvas>` — избыточна и сложнее в поддержке.

### 4. Клавиатурный обработчик — на `document`, с guard по фокусу

```js
if (['INPUT','TEXTAREA'].includes(e.target.tagName) ||
    e.target.isContentEditable) return;
```

Клавиши `[` и `]` не перехватываются YouTube-плеером, конфликтов нет.

### 5. Пересоздание контролов при навигации

В `handleNavigation()` добавляем сброс переменных и вызов `setupIntervalControls()`. Это консистентно с тем, как уже пересоздаётся `setupCopyButton()`.

## Risks / Trade-offs

- **Хрупкость селекторов YouTube** → Смягчение: используем те же классы (`.ytp-progress-bar`, `.ytp-time-display`), что и существующий код — при их изменении сломается и старый код, что будет замечено.
- **Бейдж перекрывает субтитры** → Смягчение: `pointer-events: none` для бейджа, кроме кнопок; можно добавить drag-позиционирование в будущем.
- **Прогресс-бар меняет размер** → Маркеры позиционированы через `%`, поэтому масштабируются автоматически.
