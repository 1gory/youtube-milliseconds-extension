## ADDED Requirements

### Requirement: Кнопка перехода на таймстемп
Кнопка `.ytp-jump-btn` SHALL вставляться в control bar плеера после `.ytp-copy-time-btn` (или после `.ytp-time-display` если кнопки копирования нет). На Shorts кнопка не добавляется.

#### Scenario: Кнопка появляется на обычном плеере
- **WHEN** расширение инициализируется на странице видео (не Shorts)
- **THEN** в control bar появляется кнопка `.ytp-jump-btn`

#### Scenario: Кнопка не появляется на Shorts
- **WHEN** расширение инициализируется на странице YouTube Shorts
- **THEN** кнопка не добавляется в DOM

### Requirement: Открытие поля ввода
Клик по кнопке или нажатие клавиши `g` SHALL открывать поле ввода таймстемпа поверх control bar.

#### Scenario: Открытие по клику
- **WHEN** пользователь нажимает `.ytp-jump-btn`
- **THEN** `<input class="ytp-jump-input">` появляется поверх control bar, получает фокус и выделение содержимого

#### Scenario: Открытие по клавише `g`
- **WHEN** пользователь нажимает `g` при фокусе вне полей ввода
- **THEN** выполняется то же действие, что и при клике по кнопке

#### Scenario: Клавиша `g` игнорируется при фокусе в поле ввода
- **WHEN** пользователь нажимает `g` при фокусе в `<input>`, `<textarea>` или `[contenteditable]`
- **THEN** действие не выполняется

### Requirement: Парсинг таймстемпа
`parseTimestamp(input)` SHALL принимать форматы, производимые `formatVideoTime`, и возвращать секунды (`Number`) или `null` для невалидных строк.

#### Scenario: Формат `SS.mmm`
- **WHEN** входная строка `"12.345"`
- **THEN** результат — `12.345`

#### Scenario: Формат `M:SS.mmm`
- **WHEN** входная строка `"1:30.500"`
- **THEN** результат — `90.5`

#### Scenario: Формат `H:MM:SS.mmm`
- **WHEN** входная строка `"1:02:03.004"`
- **THEN** результат — `3723.004`

#### Scenario: Формат без миллисекунд
- **WHEN** входная строка `"2:30"`
- **THEN** результат — `150`

#### Scenario: Запятая вместо точки в миллисекундах
- **WHEN** входная строка `"1:30,500"`
- **THEN** результат — `90.5`

#### Scenario: Невалидная строка
- **WHEN** входная строка `"abc"`, `""`, `"1:60:00"` или `"-1"`
- **THEN** результат — `null`

### Requirement: Применение перехода
При валидном вводе и нажатии `Enter` плеер SHALL прыгать на указанное время с clamp в `[0, video.duration]`.

#### Scenario: Валидный таймстемп внутри диапазона
- **WHEN** пользователь вводит `"1:30.500"` (видео длительностью 600 с) и нажимает `Enter`
- **THEN** `video.currentTime = 90.5`, поле закрывается

#### Scenario: Таймстемп больше длительности
- **WHEN** пользователь вводит таймстемп `"10:00"` для видео длительностью 300 с
- **THEN** `video.currentTime = video.duration` (clamp), поле закрывается

#### Scenario: Отрицательный таймстемп
- **WHEN** пользователь ввёл значение, парсящееся в отрицательное число (`parseTimestamp` это блокирует, но при обходе)
- **THEN** `video.currentTime = 0`

#### Scenario: Невалидный ввод и Enter
- **WHEN** пользователь вводит невалидную строку и нажимает `Enter`
- **THEN** поле получает класс `.ytp-jump-input--error`; `video.currentTime` не меняется; поле остаётся открытым

### Requirement: Закрытие поля ввода
Поле SHALL закрываться по `Esc`, `blur` или после успешного применения; при навигации — также закрываться.

#### Scenario: Закрытие по Esc
- **WHEN** пользователь нажимает `Esc` при открытом поле
- **THEN** поле удаляется из DOM, плеер не меняет позицию

#### Scenario: Закрытие по blur
- **WHEN** поле теряет фокус
- **THEN** поле удаляется из DOM

#### Scenario: Закрытие при навигации
- **WHEN** срабатывает `handleNavigation()` с открытым полем
- **THEN** поле удаляется до пересоздания контролов

### Requirement: Автоподстановка из буфера обмена
При открытии поля расширение SHALL пытаться прочитать буфер обмена и подставить значение, если оно парсится как таймстемп.

#### Scenario: Буфер содержит валидный таймстемп
- **WHEN** поле открывается, в буфере строка `"1:30.500"`
- **THEN** поле получает `value = "1:30.500"` и выделяет содержимое

#### Scenario: Буфер содержит невалидное значение
- **WHEN** поле открывается, в буфере произвольный текст
- **THEN** поле остаётся пустым

#### Scenario: Доступ к буферу запрещён
- **WHEN** `navigator.clipboard.readText()` отклонён или недоступен
- **THEN** ошибка перехватывается; поле открывается пустым; пользователю не показывается сообщение об ошибке
