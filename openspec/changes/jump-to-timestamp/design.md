## Context

`video.currentTime = seconds` — стандартный путь для перемотки HTML5-видео; YouTube корректно реагирует на присваивание. Формат таймстемпа, который генерирует расширение, фиксирован в `formatVideoTime(seconds, showMs)` и имеет 4 разновидности:
- `SS.mmm` (для < 60 с с миллисекундами)
- `M:SS.mmm`
- `H:MM:SS.mmm`
- `M:SS` / `H:MM:SS` (когда `showMs = false`)

Inline-инпут проще встроить в DOM плеера, чем модальное окно — он не блокирует видео и закрывается по Esc / blur.

## Goals / Non-Goals

**Goals:**
- Принимать любой формат, который расширение само производит.
- Точность до миллисекунды.
- Один шорткат — открыть поле, ввести, `Enter`.
- Безопасный clamp в `[0, video.duration]`.

**Non-Goals:**
- Парсинг произвольных пользовательских форматов (`"1m 30s"`, `"1:30 PM"`).
- Поддержка интервалов или относительных смещений (`+10s`).
- Сохранение истории таймстемпов.

## Decisions

### 1. Парсер — один общий `parseTimestamp(input): number | null`

Логика:
1. `trim()` + замена запятой на точку (для локалей).
2. Разделить по `:` — 1/2/3 части.
3. Последняя часть может содержать `.mmm`.
4. Собрать секунды: `h*3600 + m*60 + s + ms/1000`.
5. Валидация: числа неотрицательные, минуты/секунды < 60.
6. Возврат `null` для невалидных — UI отображает состояние ошибки.

### 2. Inline-инпут вместо модалки

`<input>` с `position: absolute` поверх control bar. Закрытие — `Esc`, `blur`, `Enter` (после применения). Альтернатива — модалка — отклонена: перекрывает видео, требует overlay.

### 3. Шорткат `g`

Клавиша `g` свободна в YouTube-плеере (не конфликтует со стандартными `j/k/l`, `f`, `m`, `c`). Guard по фокусу — как в `interval-timer`: игнор при фокусе в `input/textarea/[contenteditable]`.

### 4. Автоподстановка из буфера

При открытии поля — `navigator.clipboard.readText()` (если разрешено), проверка через `parseTimestamp`. Если валидно — подставить в `value` и выделить весь текст. При отказе в доступе — поле остаётся пустым (не ошибка).

### 5. Clamp в диапазон

`seconds = Math.max(0, Math.min(seconds, video.duration))`. При превышении длительности — переход в конец, не ошибка: пользовательский UX мягче.

## Risks / Trade-offs

- **`clipboard.readText()` требует фокус и разрешение** → Смягчение: graceful — если отказ, продолжаем с пустым полем; не показываем ошибку.
- **Конфликт `g` с расширениями браузера** → Малый риск: `g` обычно свободна; если возникнет, можно сделать настраиваемой через popup.
- **Парсер пропускает edge-cases** (например, `60:00`) → Валидация `< 60` для не-самой-старшей части; самая старшая (часы/минуты при отсутствии часов) — без верхней границы.
- **YouTube перехватывает `g` на каких-то страницах** → Тест на странице видео; при перехвате — `e.stopPropagation()` в обработчике расширения, ставим listener в capture-фазе.
